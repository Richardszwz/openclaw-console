# 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
OpenClaw Console - 智能体统一管理平台

### 1.2 目标用户
- 企业技术总监/CTO
- AI 系统管理员
- DevOps 工程师
- 技术团队负责人

### 1.3 产品目标
打造一个比原版 Control Center 更强大、更稳定、更易用的 OpenClaw 管理控制台，解决原版内存泄漏、功能有限的问题。

---

## 2. 功能需求

### 2.1 Gateway Hub - 网关监控中心

#### 2.1.1 仪表盘
- **实时状态卡片**: 显示 Gateway 运行状态、版本、运行时长
- **性能指标**: QPS、延迟、内存占用、连接数
- **渠道状态**: 微信、飞书等渠道连接状态
- **告警摘要**: 显示最近的告警信息

#### 2.1.2 日志中心
- **实时日志流**: 类似 tail -f 的实时日志
- **搜索过滤**: 按关键字、时间、级别筛选
- **日志导出**: 支持导出为文件

#### 2.1.3 操作面板
- **启动/停止/重启**: 一键控制 Gateway
- **配置热更新**: 修改配置后无需重启
- **紧急模式**: 快速停止所有会话

### 2.2 Agent Fleet - 智能体舰队

#### 2.2.1 Agent 列表
- **卡片视图**: 每个 Agent 显示名称、状态、模型
- **表格视图**: 详细信息的表格展示
- **状态筛选**: 按运行状态筛选

#### 2.2.2 Agent 详情
- **基本信息**: 工作目录、模型配置
- **技能列表**: 显示已启用的技能
- **性能统计**: 会话数、Token 使用量
- **最近活动**: 最近处理的会话

#### 2.2.3 Agent 操作
- **重启 Agent**: 重新加载配置
- **编辑配置**: 修改 skills、模型等
- **查看日志**: 查看该 Agent 的专属日志

### 2.3 Session Manager - 会话管理中心

#### 2.3.1 会话列表
- **实时更新**: WebSocket 推送状态变化
- **多维度筛选**: 按 Agent、渠道、状态筛选
- **搜索功能**: 按会话 ID 或内容搜索
- **批量操作**: 支持多选后批量 Kill/Delete

#### 2.3.2 会话详情
- **对话历史**: 查看完整的对话记录
- **Token 统计**: 输入/输出 Token 数
- **耗时分析**: 各阶段耗时统计
- **关联会话**: 显示子会话（Subagent）

#### 2.3.3 会话操作
- **Kill**: 终止运行中的会话
- **Delete**: 删除会话记录
- **Export**: 导出会话内容为 JSON

### 2.4 Workflow Studio - 工作流工作室

#### 2.4.1 工作流列表
- **标准模板**: 六/十 Agent 标准流程
- **自定义流程**: 用户创建的工作流
- **版本管理**: 工作流版本历史

#### 2.4.2 流程设计器
- **拖拽画布**: 拖拽节点设计流程
- **节点类型**: Agent 节点、审批节点、条件节点
- **连线编辑**: 连接节点形成流程
- **属性面板**: 编辑节点配置

#### 2.4.3 流程监控
- **实时状态**: 显示当前运行的任务
- **进度追踪**: 显示任务在各节点的进度
- **历史回放**: 查看历史流程执行

### 2.5 Task Scheduler - 任务调度中心

#### 2.5.1 定时任务列表
- **Cron 表达式**: 可视化编辑 Cron
- **执行历史**: 最近 50 次执行记录
- **失败告警**: 失败任务红色标记

#### 2.5.2 任务编辑器
- **自然语言**: "每天凌晨 3 点"
- **Cron 验证**: 实时验证 Cron 有效性
- **下次执行**: 显示下次执行时间

#### 2.5.3 执行监控
- **实时日志**: 任务执行实时输出
- **耗时统计**: 历史执行耗时趋势
- **失败重试**: 配置重试策略

### 2.6 Model Usage Dashboard - 模型用量监控

#### 2.6.1 模型配置
- **多平台支持**: Volcengine (Doubao), MiniMax, OpenAI, Anthropic, Google Gemini
- **配额配置**: 月度总量配额、每日配额
- **价格配置**: 按每百万 tokens 计费
- **自动同步**: 从 openclaw.json 读取配置

#### 2.6.2 用量统计
- **今日用量**: 显示当日输入/输出 tokens
- **本月用量**: 累计月度用量
- **用量占比**: 显示配额使用百分比
- **成本估算**: 根据价格计算估算成本

#### 2.6.3 趋势图表
- 日用量趋势图（近30天）
- 模型用量对比饼图
- 成本趋势折线图

#### 2.6.4 总量预警
- 可配置预警阈值（如 80%）
- 用量超过阈值自动告警
- 支持飞书/邮件通知
- 告警记录查询

### 2.7 Settings Center - 配置中心

#### 2.7.1 OpenClaw 配置
- **JSON 编辑器**: 带语法高亮的配置编辑
- **配置验证**: 保存前验证 JSON 格式
- **配置备份**: 自动备份历史配置

#### 2.7.2 模型配置
- **模型列表**: 管理所有 LLM 模型
- **API Key**: 安全存储 API 密钥
- ** fallback**: 配置模型降级策略
- **用量预警配置**: 配置阈值和通知方式

#### 2.7.3 权限管理
- **用户管理**: 多用户支持
- **角色权限**: 管理员/只读/操作员
- **操作审计**: 记录所有管理操作

## 3. 非功能需求

### 3.1 性能要求
- 仪表盘加载时间 < 2 秒
- 会话列表支持 1000+ 条记录流畅滚动
- WebSocket 延迟 < 100ms
- 内存占用 < 500MB（比原版 Control Center 大幅降低）

### 3.2 可靠性要求
- 支持断线重连
- 关键操作二次确认
- 数据自动备份
- 崩溃自动恢复

### 3.3 安全要求
- Token 认证
- 操作审计日志
- 敏感信息脱敏显示
- CSRF 防护

### 3.4 兼容性要求
- 支持 Chrome/Edge/Firefox/Safari 最新版
- 响应式设计，支持平板访问
- 支持深色模式

## 4. UI/UX 设计规范

### 4.1 设计原则
- **清晰**: 信息层次分明
- **高效**: 常用操作一键可达
- **实时**: 状态变化即时反馈
- **安全**: 危险操作明确提示

### 4.2 色彩规范
- 主色: #1677FF (Ant Design Blue)
- 成功: #52C41A
- 警告: #FAAD14
- 错误: #F5222D
- 背景: #F0F2F5

### 4.3 布局规范
- 左侧导航栏固定 200px
- 顶部 Header 64px
- 内容区自适应
- 最大宽度 1920px

## 5. 里程碑与验收标准

### Phase 1: MVP (Week 1-4)
**目标**: 核心监控功能可用

**验收标准**:
- [ ] Gateway 状态仪表盘可正常显示
- [ ] Agent 列表显示正确
- [ ] 会话列表支持 Kill/Delete
- [ ] 基本配置可修改

### Phase 2: 核心功能 (Week 5-8)
**目标**: 完整核心功能可用

**验收标准**:
- [ ] 工作流可视化完成
- [ ] 定时任务管理完成
- [ ] 日志搜索功能可用
- [ ] 告警通知功能可用

### Phase 3: 扩展功能 (Week 9-12)
**目标**: 模型用量监控

**验收标准**:
- [ ] 多平台模型用量统计
- [ ] 用量趋势图表
- [ ] 配额超限预警
- [ ] 飞书通知告警

**目标**: 生产就绪

**验收标准**:
- [ ] 多用户权限管理
- [ ] 数据导出报表

- [ ] 文档完善

## 6. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| OpenClaw API 变更 | 高 | 封装 API 层，隔离变化 |
| 内存泄漏 | 高 | 严格测试，使用虚拟列表 |
| WebSocket 不稳定 | 中 | 实现断线重连机制 |
| 功能过于复杂 | 中 | 按 MVP 迭代，控制范围 |
