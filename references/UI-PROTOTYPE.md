## 3. 操作流程图

### 3.1 用户登录流程

```
用户打开 Console
    │
    ▼
检查 Gateway 状态
    │
    ├─ 运行中 ──► 显示仪表盘
    │
    └─ 未运行 ──► 提示启动 Gateway
                      │
                      ▼
                [启动 Gateway]
                      │
                      ▼
                启动成功? ──┬─ 是 ──► 显示仪表盘
                      │    │
                      否   │
                      │    │
                      ▼    │
                显示错误信息
```

### 3.2 查看 Agent 状态流程

```
用户点击 "Agent舰队"
    │
    ▼
读取 openclaw.json 配置
    │
    ▼
获取每个 Agent 的状态
    │
    ▼
渲染 Agent 卡片列表
    │
    ▼
用户点击某个 Agent
    │
    ▼
显示 Agent 详情页
    │
    ├─ 查看配置
    ├─ 查看日志
    └─ [重启 Agent]
            │
            ▼
      发送重启命令
            │
            ▼
      刷新状态
```

### 3.3 Kill 会话流程

```
用户进入 "会话管理"
    │
    ▼
显示会话列表
    │
    ▼
用户选择会话
    │
    ▼
点击 [Kill] 按钮
    │
    ▼
确认对话框: "确定要终止此会话?"
    │
    ├─ 取消 ──► 返回列表
    │
    └─ 确认
          │
          ▼
    发送 Kill 命令到 Gateway
          │
          ▼
    等待响应
          │
          ├─ 成功 ──► 更新状态为 "killed"
          │
          └─ 失败 ──► 显示错误信息
```

### 3.4 创建工作流流程

```
用户进入 "工作流"
    │
    ▼
点击 [新建工作流]
    │
    ▼
输入工作流名称
    │
    ▼
进入流程设计器
    │
    ▼
拖拽组件到画布:
    - 开始节点
    - Agent 节点
    - 审批节点
    - 结束节点
    │
    ▼
连接节点形成流程
    │
    ▼
配置每个节点属性
    │
    ▼
点击 [保存]
    │
    ▼
验证流程完整性
    │
    ├─ 验证通过 ──► 保存成功
    │
    └─ 验证失败 ──► 提示错误
```

### 3.5 告警处理流程

```
系统检测到异常
    │
    ▼
生成告警事件
    │
    ├─ 写入数据库
    ├─ WebSocket 推送
    └─ 发送通知(飞书/邮件)
    │
    ▼
仪表盘显示告警 badge
    │
    ▼
用户查看告警列表
    │
    ▼
用户处理告警
    │
    ├─ [标记已处理]
    ├─ [忽略]
    └─ [查看详情]
```

## 4. 设计原则

### 4.1 一致性
- 所有按钮使用 Ant Design 组件
- 统一的颜色规范
- 统一的间距和字体

### 4.2 反馈
- 操作后立即有视觉反馈
- 耗时操作显示 loading
- 错误信息明确提示

### 4.3 效率
- 常用操作一键可达
- 批量操作支持
- 快捷键支持

### 4.4 安全
- 危险操作二次确认
- 敏感信息脱敏显示
- 操作审计记录
